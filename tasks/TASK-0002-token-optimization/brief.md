# 简介

## 用户诉求

用户反馈当前多 Agent 协作流程体系在使用时 token 消耗过大，希望根据已讨论的优化方向调整规范。

## 背景

现有规范默认进入完整多 Agent 流程，并要求每个执行任务建立多份阶段文档。该设计有利于高风险协作，但在问答、小修复、纯文档维护和单 owner 任务上容易产生重复上下文、重复扫描和重复阶段记录。

## 目标

- 将默认执行策略从“全量流程优先”调整为“按风险分级升级”。
- 引入项目上下文基线和增量 Discovery，减少重复扫描与重复复述。
- 允许低风险任务使用轻量记录，同时保留高风险任务的完整质量门禁。
- 约束阶段输出格式，避免阶段文档互相复制完整上下文。

## 范围

范围内：

- `AGENTS.md`
- `.agents/config.yaml`
- `.agents/workflows/feature-development.yaml`
- 必要的 `.agents/agents/*.md` 角色规则
- 本任务目录下的任务记录

范围外：

- 业务代码
- 前端、后端、数据、基础设施实现
- 项目真实业务模块的 ownership 调整
