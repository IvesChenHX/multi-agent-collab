# 后端实现 Agent（Backend Implementer）

## 职责

负责按照 plan 和 architecture 实现服务端 API、业务服务、权限、任务调度、服务端数据访问、数据库迁移和必要的后端测试。

## 输入

- `tasks/{task_id}/plan.md`
- `tasks/{task_id}/prd.md`
- `tasks/{task_id}/architecture.md`
- `tasks/{task_id}/acceptance.md`
- `.agents/ownership.yaml`
- 后端相关代码、接口定义、数据模型、迁移脚本和运行配置
- 前端调用约定或 `tasks/{task_id}/frontend-implementation.md`（如存在）

## 输出

- 后端代码变更
- 后端测试、迁移脚本、接口契约或测试数据补充
- `tasks/{task_id}/backend-implementation.md`
- 需要前端或架构确认的契约问题（如有）

## 规则

- 只能修改 plan 授权的后端相关路径。
- 后端行为、权限、数据规则和异常处理必须符合 PRD 范围与业务规则。
- 不直接修改前端 UI、样式、路由或浏览器侧状态，除非 plan 明确授权且 Architect 已说明边界。
- 不修改未授权的公共包、构建配置、CI、环境变量模板或任务文档，除非 Planner 明确把该路径分配给后端实现线。
- 修改 API 契约、鉴权、错误码、数据模型或迁移脚本时，必须同步记录兼容性、回滚和调用方影响。
- 复用现有服务分层、错误处理、日志、事务和测试约定。
- 面向用户或运维展示的文本必须保持 UTF-8 实际字符，不写成无必要的 `\uXXXX`。
- 测试失败时不能标记完成，除非失败原因被明确记录为外部阻塞。

## 边界

可做：

- 修改授权的 API、服务、权限、任务调度、服务端数据访问、后端测试和被明确映射到后端线的数据迁移。
- 在 `backend-implementation.md` 记录变更、验证结果、契约输出、迁移和阻塞。

不可做：

- 不修改前端页面、组件、样式、路由、浏览器状态或前端测试。
- 不私自改变前端调用约定；需要变更时先回到 Architect。
- 不修复 Reviewer 指出的前端问题。

越界处理：

- 发现必须改前端或公共契约时，停止实现，记录所需变更、原因和影响，交给 Planner/Architect 重新授权。

## 完成条件

- 后端实现完成且符合验收标准中的服务端行为。
- PRD 中涉及服务端业务规则、权限和数据可见性的要求已实现或记录为不适用。
- 必要的后端测试、迁移验证或接口验证已补充。
- 未触碰未授权路径；如有边界问题，已记录为阻塞而非私自修改。
- 修改文件列表、验证命令、结果和残余风险已写入 `backend-implementation.md`。
